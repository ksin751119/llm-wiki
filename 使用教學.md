## 1. 你需要什麼

開始之前，確認你有以下工具：

| 工具 | 用途 | 安裝方式 |
|------|------|---------|
| **Claude Code** | LLM 代理人，幫你維護 wiki | 已安裝（你正在用它） |
| **Obsidian** | 瀏覽和查看 wiki | 下一步安裝 |
| **Git** | 版本控制（選用但推薦） | 大部分系統已內建 |

---

## 2. 安裝 Obsidian

1. 前往 https://obsidian.md
2. 下載對應你作業系統的版本（Windows / Mac / Linux）
3. 安裝並打開

打開後先不用做任何事，等我們建好 wiki 資料夾再回來。

---

## 3. 設定 Obsidian 插件

打開 Obsidian 後，安裝以下插件。每個插件的安裝方式都一樣：

> **安裝插件的通用步驟：**
> 1. 點左下角的 ⚙️（Settings）
> 2. 左邊選單找到 **Community plugins**
> 3. 如果是第一次，點 **Turn on community plugins**
> 4. 點 **Browse**
> 5. 搜尋插件名稱 → 點 **Install** → 點 **Enable**

### 必裝插件

#### Obsidian Web Clipper（瀏覽器插件）

這不是 Obsidian 內建插件，是瀏覽器擴充功能：

1. 用瀏覽器前往 https://obsidian.md/clipper
2. 點 **Get the extension**（Chrome / Firefox / Safari 都有）
3. 安裝到你的瀏覽器
4. 安裝完成後，瀏覽器右上角會出現 Obsidian 圖示

**用途：** 看到一篇好文章 → 點一下 → 自動存成 markdown 到你的 wiki。

### 推薦插件

| 插件名稱 | 用途 | 必要性 |
|----------|------|--------|
| **Dataview** | 對 wiki 頁面做動態查詢和表格 | 推薦 |
| **Marp Slides** | 把 markdown 頁面變成簡報 | 選用 |

### 設定圖片自動下載

這一步很重要——讓文章中的網路圖片自動下載到本地，避免日後連結失效。

1. 打開 Settings（⚙️）
2. 左邊選 **Files and links**
3. 找到 **Attachment folder path** → 改成 `raw/assets`
4. 左邊選 **Hotkeys**
5. 搜尋框輸入 `Download`
6. 找到 **Download attachments for current file**
7. 點右邊的 `+` 設定快捷鍵 → 按 `Ctrl+Shift+D`（Mac: `Cmd+Shift+D`）

**以後用法：** Web Clipper 存文章後 → 在 Obsidian 打開那篇文章 → 按 `Ctrl+Shift+D` → 所有圖片自動下載到本地。

## 4. 安裝 llm-wiki-guide 技能

打開終端機，執行：

```bash
unzip /home/ubuntu/DEV/llm-wiki/llm-wiki-guide.skill -d ~/.claude/skills/llm-wiki-guide
```

重新啟動 Claude Code，技能就會自動載入。

**驗證安裝成功：** 在 Claude Code 中輸入 `/llm-wiki-guide`，如果看到選單就表示安裝成功。

---

## 5. 建立你的第一個 Wiki

在 Claude Code 中輸入：

```
/llm-wiki-guide:setup
```

Claude 會問你幾個問題：

```
Claude: "What topic/domain is this wiki for?"
你:     "咖啡研究"（或任何你想研究的主題）

Claude: "What language should the wiki be in?"
你:     "繁體中文"

Claude: "Where should the wiki live?"
你:     按 Enter 用預設路徑，或輸入你想要的路徑
```

完成後，Claude 會自動幫你：
- 建立 `raw/`（放原始來源）和 `wiki/`（放 wiki 頁面）資料夾
- 產生 `CLAUDE.md`（Schema，定義 wiki 的規則）
- 產生空的 `wiki/index.md` 和 `wiki/log.md`
- 執行 `git init`

---

## 6. 用 Obsidian 打開 Wiki

1. 打開 Obsidian
2. 點 **Open folder as vault**（或 Open → Open folder as vault）
3. 選擇剛才建立的 wiki 資料夾（例如 `~/wiki-咖啡研究`）
4. Obsidian 會打開這個資料夾，你可以在左邊看到檔案結構

**打開 Graph View：**
1. 按 `Ctrl+G`（Mac: `Cmd+G`）
2. 或是點左邊工具列的「分子圖」圖示

現在圖裡還沒什麼東西，等你加入幾個來源後就會看到頁面之間的連結。

## 7. 加入第一個來源

### 方法 A：用 Web Clipper 存網路文章

1. 在瀏覽器打開你想收藏的文章
2. 點瀏覽器右上角的 Obsidian 圖示
3. 存檔位置選 `raw/` 資料夾
4. 點 **Add to Obsidian**
5. 回到 Obsidian，打開那篇文章 → 按 `Ctrl+Shift+D` 下載圖片

### 方法 B：手動放入檔案

把 PDF、markdown、或文字檔直接拖進 `raw/` 資料夾。

### 讓 Claude 處理來源

在 Claude Code 中輸入：

```
/llm-wiki-guide:ingest
```

Claude 會：
1. 列出 `raw/` 裡的檔案，問你要處理哪個
2. 讀取全文，告訴你重點摘要
3. 問你有沒有想強調或跳過的部分
4. 自動建立摘要頁、更新相關頁面、更新索引

**完成後，切換到 Obsidian 看看：**
- `wiki/` 裡多了新頁面
- `wiki/index.md` 已更新
- 按 `Ctrl+G` 看 Graph View，開始出現連結了！

### 重複這個步驟

前 5 個來源建議逐一 ingest（每次都和 Claude 討論）。之後如果想快一點，可以告訴 Claude「直接處理，不用討論」。

## 8. 向 Wiki 提問

加入幾個來源後，你可以開始問問題。在 Claude Code 中輸入：

```
/llm-wiki-guide:query
```

然後問你的問題，例如：
- 「幫我比較 A 和 B」
- 「目前 wiki 裡關於 X 的知識有哪些？」
- 「有沒有什麼矛盾的地方？」

Claude 會讀 wiki 頁面，綜合出答案。**答案結束後 Claude 會問你：**

> "This answer might be useful for future reference. Want me to save it as a wiki page?"

如果你覺得這個回答不錯，就說「好」——它就會存成一個 wiki 頁面，以後可以直接引用。

## 9. 健康檢查

當你收到 Claude 的提醒「Consider running `/llm-wiki-guide:lint`」，或你自己想檢查一下，輸入：

```
/llm-wiki-guide:lint
```

Claude 會幫你檢查：
- 有沒有互相矛盾的內容
- 有沒有孤立的頁面（沒人連過去的）
- 有沒有該建但還沒建的頁面
- 有沒有過時的資訊
- 交叉引用是否完整
- 還缺什麼來源

檢查完後 Claude 會給你一份報告，告訴你哪些問題已自動修復、哪些需要你決定。

## 10. 日常使用流程

每天使用其實就是重複這幾個動作：

```
找到好文章
    │
    ▼
Web Clipper 存到 raw/         ← 在瀏覽器點一下
    │
    ▼
/llm-wiki-guide:ingest        ← 在 Claude Code 輸入
    │
    ▼
Obsidian 瀏覽更新的 wiki      ← 切換到 Obsidian 看結果
    │
    ▼
有問題就 /llm-wiki-guide:query
    │
    ▼
每 5 次 ingest 做一次 /llm-wiki-guide:lint
```

### 隨時可用的指令速查

| 你想做什麼 | 輸入什麼 |
|------------|---------|
| 不知道該做什麼 | `/llm-wiki-guide` |
| 加入新來源 | `/llm-wiki-guide:ingest` |
| 問問題 | `/llm-wiki-guide:query` |
| 健康檢查 | `/llm-wiki-guide:lint` |
| 看 wiki 狀態 | `/llm-wiki-guide:status` |

### 小技巧

- **Graph View 常開著**：隨時按 `Ctrl+G` 看 wiki 的知識結構，哪裡密集（研究得多）、哪裡空白（需要更多來源）
- **不用記得 lint**：Claude 會自動提醒你
- **有價值的問答都存回 wiki**：這是「知識複利」的關鍵——問得越多，wiki 越豐富，以後的答案品質越高
- **Schema 隨時可調**：用了一陣子覺得某些慣例不順，直接告訴 Claude「更新 Schema」就好
