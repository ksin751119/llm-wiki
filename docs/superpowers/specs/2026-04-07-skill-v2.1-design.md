# LLM Wiki Guide Skill v2.1 — Design Spec

> Date: 2026-04-07
> Status: Draft
> Scope: 7 項修正，基於 skill-creator 原則審查

## Background

v2 完成後，以 skill-creator 的核心原則（Concise is Key、Progressive Disclosure、Degrees of Freedom）審查現有 skill，發現 7 個可優化項目。本次為純修正，不加新功能。

## Fix 1: 刪除 SKILL.md Smart Reminders，改指向 reference

### 問題

SKILL.md 的 Smart Reminders 表（10 行）與 ingest-guide 和 query-guide 的 Post-* Reminders 高度重疊，浪費 context window。

### 設計

**SKILL.md：** 刪除整個 `## Smart Reminders` 區段（lines 95-110，含表頭和 10 行表格），替換為：

```markdown
## Reminders

Each sub-command guide includes context-aware reminders (post-ingest, post-query, post-lint). Follow the reminders in the active guide.
```

**lint-guide.md：** 現有 Post-Lint 區段只有「fix → log → suggest next actions」。在 item 3 (`Suggest next actions based on findings`) 之後加入具體 reminders：

```markdown
4. Specific reminders:
   - If many data gaps found: "Consider finding new sources to fill these gaps"
   - If all checks passed: "Wiki is healthy. Continue ingesting sources or explore with `/llm-wiki-guide:query`"
   - If contradictions found: "Review the contradictions above and decide how to resolve them"
```

ingest-guide 和 query-guide 的 Post-* Reminders 保持不動（它們已經完整）。

## Fix 2: Schema Query Workflow 更新

### 問題

setup-guide.md 的 Schema 模板中 Query Workflow 仍是 v1 的 4 步流程，沒有反映 v2 的探索能力。使用者建立的 wiki CLAUDE.md 會包含過時描述。

### 設計

替換 setup-guide.md Schema 模板中的 `## Query Workflow` 區段（lines 104-109）為：

```markdown
## Query Workflow
When asked a question:
1. Read `wiki/index.md` to find relevant pages
2. Read those pages
3. Synthesize an answer with citations to wiki pages
4. Offer to explore deeper (multi-turn dialogue citing wiki pages)
5. If explored, synthesize insights and collect any contradictions into Tensions
6. Ask if answer/insights should be saved as a new wiki page
```

## Fix 3: Frontmatter description 加入 v2 關鍵字

### 問題

SKILL.md frontmatter description 仍是 v1 版本，缺少探索、Tensions、互動原則等 v2 新功能關鍵字，影響 skill 觸發準確度。

### 設計

替換 SKILL.md 的 description 為：

```yaml
description: "Guide users through building and maintaining a personal LLM Wiki — a persistent, LLM-maintained knowledge base of interlinked markdown files. Sub-commands: setup, ingest, query (with deep exploration and insight sedimentation), lint (including tensions audit), status. Use when: user wants to create a knowledge base, build a wiki with LLM, organize research/notes/reading with AI, explore a topic against existing knowledge, track contradictions across sources, or says 'llm-wiki', 'knowledge base', 'wiki setup', 'ingest source', 'explore topic', 'knowledge tensions'."
```

變更重點：
- query 後面加了 `(with deep exploration and insight sedimentation)`
- lint 後面加了 `(including tensions audit)`
- Use when 加了 `explore a topic against existing knowledge`, `track contradictions across sources`
- 觸發詞加了 `'explore topic'`, `'knowledge tensions'`

## Fix 4: 新增 page-templates.md

### 問題

Skill 描述了 6 種頁面類型和各自的結構要求，但沒有提供具體模板。LLM 每次都要從 writing-standards 的規則重新推導頁面結構。

### 設計

**新增 `references/page-templates.md`：**

```markdown
# Page Templates

Skeleton templates for wiki page types. Adapt to the wiki's domain and conventions.

## Summary Page

```markdown
{One-line summary of the source.}

**Source:** {title} by {author}, {date}
**URL:** {url, if applicable}

## Key Takeaways

### {Theme 1}

{2-3 sentences}

### {Theme 2}

{2-3 sentences}

## Notable Quotes

> "{quote}" — {attribution}

## Cross-References

- [Related Page](related-page.md) — {why related}
```

Target: 20-40 lines.

## Concept Page

```markdown
{One-line definition of the concept.}

## Overview

{2-3 paragraphs: what it is, why it matters, key dimensions.}

## Key Points

### {Aspect 1}

{Evidence and perspectives from multiple sources.}

### {Aspect 2}

{Evidence and perspectives from multiple sources.}

## Tensions & Open Questions

*No known tensions at this time.*

## Cross-References

- [Related Page](related-page.md) — {why related}
```

Target: 40-100 lines.

## Entity Page

```markdown
{One-line description of the entity.}

## Overview

{Who/what, key facts, significance.}

## Key Details

### {Dimension 1}

{Facts from sources, with attribution.}

### {Dimension 2}

{Facts from sources, with attribution.}

## Cross-References

- [Related Page](related-page.md) — {why related}
```

Target: 30-80 lines.
```

只提供 summary、concept、entity 三個最常用類型的模板。comparison、synthesis、overview 類型變化太大，硬給模板反而限制自由度。

**SKILL.md 變更：** 在 Writing Standards 段落後加一行：

```markdown
For page structure templates, see [references/page-templates.md](references/page-templates.md).
```

**ingest-guide.md 變更：** 在 Step 4 (Create Summary Page) 的第一行後加：

```markdown
Use the template in [page-templates.md](page-templates.md) as starting structure.
```

## Fix 7: status 命令改為描述性指令

### 問題

SKILL.md 的 status 區段使用硬編 shell 命令（`ls wiki/`、`grep -c`），在不同環境可能失效。

### 設計

替換 SKILL.md status 區段的步驟 1-2 為描述性指令：

```markdown
1. Count the number of wiki pages and log entries
2. Read the last 5 log entries for recent activity
3. Read index.md for coverage overview
```

其餘步驟（4. Report 和 5. Suggest next action）保持不動。
