## Tensions & Open Questions Format

Concept pages collect contradictions and unresolved questions found across sources. Each entry follows this format:

- **{Brief description of tension}** — {Source A} claims X, but {Source B} claims Y.

- One tension or one open question per bullet — do not mix
- Every entry must include source links
- Remove resolved tensions (resolution recorded in log.md)
- Tensions triggered by: ingest (new source conflicts with existing), query exploration (contradictions found in discussion), lint (inconsistencies detected in scan)

### Task 4: Update setup-guide.md — Add Wiki Maintenance Principles + concept page Tensions mention

**Files:**

- [ ] **Step 1: Add Wiki Maintenance Principles to Schema template**

In `references/setup-guide.md`, find the Schema template in Step 4. Locate the `## Conventions` section (starting with `- All cross-references use markdown links`). After the last convention line (`- Language: {language}`), and before the closing triple-backtick of the template, add:

```markdown

## Wiki Maintenance Principles

1. **Ask when contradictions arise — never silently overwrite**
   When a new source conflicts with existing wiki content, mark it with
   `> **Contradiction:**` and collect it into the relevant concept page's
   Tensions & Open Questions section. If you cannot determine which side
   is correct, ask the user.

2. **Mark uncertainty**
   Claims supported by only a single source use attribution: "According to
   {source}, ..." Do not state them as established fact. Only use assertion
   form for claims cross-verified by multiple sources.

3. **Clarify before assuming during ingest discussion**
   When discussing a source's key points with the user, if you are unsure
   what the user wants to emphasize or their stance on a claim, ask.
   Do not assume the user's position.
```

### Task 5: Rewrite query-guide.md — Upgrade from 5 steps to 7 steps

**Files:**
- Modify: `/tmp/llm-wiki-skill/llm-wiki-guide/references/query-guide.md`

This is the largest change. The current file has: Pre-check, Step 1-4 (unchanged), Step 5 (Offer to File Back), Post-Query Reminders. We keep Steps 1-4 intact, replace Step 5, and add Steps 5-7 plus updated reminders.

- [ ] **Step 1: Replace Step 5 with new Steps 5-7**

In `references/query-guide.md`, find the section `## Step 5: Offer to File Back` and everything after it (through the end of the file). Replace it entirely with:

```markdown
## Step 5: Offer to Explore Deeper

After presenting the answer, ask:

> "Want to explore this further? I can dig deeper using what's in the wiki."

- If the user declines → skip to Step 7
- If the user continues → enter exploration mode:
  - **Cite wiki pages** as discussion material: "concept-xxx.md mentions... what's your take?"
  - **Flag contradictions** when found: "This seems to conflict with what's in entity-yyy.md"
  - **Ask follow-up questions** proactively — don't just answer, challenge and probe
  - Continue until the user is satisfied or the conversation naturally concludes

## Step 6: Synthesize Insights

After exploration ends (only triggered if user entered exploration in Step 5):

1. Summarize 3-5 key insights from the discussion (one sentence each)
2. Mark which insights contradict existing wiki content
3. Present to the user for confirmation

## Step 7: File Back to Wiki

**If no exploration happened (user declined at Step 5):**

Ask:

> "This answer might be useful for future reference. Want me to save it as a wiki page?"

If yes:
1. Choose page type (usually `synthesis-*` or `compare-*`)
2. Write the page to `wiki/`, following [writing-standards.md](writing-standards.md)
3. Update `wiki/index.md`
4. Append to `wiki/log.md`:
   ```
   ## [{date}] query | {question summary}
   - Generated {page-name}.md
   - Filed as permanent wiki page
   ```

If no, move on.

**If exploration happened (user went through Steps 5-6):**

Ask:

> "Want to save these insights as a wiki page?"

If yes:
1. Write the page (usually `synthesis-*`), following [writing-standards.md](writing-standards.md)
2. Update `wiki/index.md`, `wiki/log.md`, `wiki/_backlinks.json`
3. If insights contain contradictions, collect them into the `## Tensions & Open Questions` section of relevant concept pages (see [writing-standards.md](writing-standards.md) for format)
4. Log format: `## [{date}] query+explore | {topic summary}`

If no:
1. If contradictions were found during exploration, still collect them into relevant concept pages' Tensions sections — contradiction tracking does not depend on whether the user chooses to save a page
2. Log format: `## [{date}] query+explore | {topic summary} (not filed)`

## Post-Query Reminders

- If query revealed gaps: "Your wiki could benefit from sources about {topic}. Want to find some?"
- If query referenced stale pages: "Some pages haven't been updated recently. Consider `/llm-wiki-guide:lint`."
- If exploration produced contradictions: "Found {N} contradictions. Consider running `/llm-wiki-guide:lint` to review them."
- If exploration happened but user chose not to save: "No problem. You can always revisit this topic with another query later."
```
