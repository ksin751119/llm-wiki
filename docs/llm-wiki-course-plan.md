## 各章節詳細計畫

### 第一章：總覽與定位
**重點：**
1. RAG 的問題：無累積、每次從零開始
2. LLM Wiki 的核心差異：持久化編譯產物
3. 人 vs LLM 的角色分工

**理解確認：**
- Q: Wiki 模式和 RAG 模式的根本差異是什麼？
- Q: 在 LLM Wiki 中，人類和 LLM 各自負責什麼？

### 第二章：架構全景
**重點：**
1. 三層架構及其擁有者
2. Raw Sources 的不可變性
3. Schema 的約束角色
4. 三大操作的位置

**理解確認：**
- Q: 為什麼 Raw Sources 必須是不可變的？
- Q: Schema 層如何影響 LLM 的行為？

### 第三章：核心資料結構
**重點：**
1. 六種 wiki 頁面類型
2. index.md 的結構與用途
3. log.md 的設計與 index.md 的差異（空間 vs 時間）

**理解確認：**
- Q: index.md 和 log.md 各自在什麼場景被使用？
- Q: 為什麼需要兩個不同的索引？

### 第四章：核心流程 — Ingest
**重點：**
1. Ingest 的 6 個步驟
2. 級聯更新（一個來源觸及 10-15 頁）
3. 使用者參與的價值
4. 單筆 vs 批量 Ingest 的取捨

**理解確認：**
- Q: 為什麼一個來源的 Ingest 會觸及 10-15 個頁面？
- Q: 逐一 Ingest 比批量 Ingest 好在哪裡？

### 第五章：進階流程 — Query 與 Lint
**重點：**
1. Query 的知識複利概念
2. 答案回填 wiki 的判斷
3. Lint 的四大檢查項目與主動建議

**理解確認：**
- Q: 什麼是「知識複利」？為什麼 Query 結果應該回填 wiki？
- Q: Lint 除了修復問題，還做什麼？
